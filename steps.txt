Ini step list bangun proyek Sistem Perpustakaan Kampus dari awal, sesuai PRD final (dengan tech stack di section 7.1).

## Fase 0 — Perencanaan
1. PRD sudah final (selesai)
2. Buat SRS (breakdown requirement teknis per modul + mapping ke 7 unit kompetensi LSP)
3. Rancang ERD di dbdiagram.io — entitas: anggota, buku, kategori_buku, peminjaman, pengembalian, dokumen, users (Administrator/Petugas)
4. Export DBML dari dbdiagram.io → SQL sebagai referensi DDL

## Fase 1 — Setup Database
5. Install MySQL/MariaDB lokal (kalau belum ada)
6. Buat database baru (`perpustakaan_db` atau sejenisnya)
7. Jalankan DDL — buat semua tabel sesuai ERD
8. Buat seed data awal (contoh anggota, buku, kategori)

## Fase 2 — Kualitas Data & Integrasi CSV
9. Siapkan sample data kotor sengaja (data ganda, kosong, format salah, stok tidak valid)
10. Tulis query/script identifikasi data bermasalah
11. Tulis query/script perbaikan data
12. Buat laporan before/after kualitas data
13. Siapkan file CSV data anggota
14. Buat script import CSV ke database (Node.js/`mysql2` atau tools sejenis)
15. Buat mekanisme verifikasi anti-duplikat saat re-import

## Fase 3 — Logika Basis Data
16. Buat views untuk laporan (buku tersedia/dipinjam, riwayat peminjaman, dsb)
17. Buat stored procedures/triggers (misal: update status buku otomatis saat peminjaman/pengembalian)
18. Tulis & uji query SQL: INSERT, SELECT, UPDATE, DELETE, JOIN, agregasi, pencarian

## Fase 4 — Dokumen & Akses
19. Buat tabel/struktur metadata dokumen
20. Input contoh dokumen dengan metadata lengkap
21. Buat akun database untuk role Administrator dan Petugas (GRANT/REVOKE)
22. Uji perbedaan hak akses antar role (buktikan satu role ditolak saat akses di luar kewenangannya)
23. Buat matriks hak akses (dokumentasi visual)

## Fase 5 — Backend
24. Init project Express + install `mysql2`
25. Buat koneksi database (`db.js`)
26. Buat endpoint login (bedakan role Administrator/Petugas)
27. Buat endpoint CRUD anggota, buku, kategori
28. Buat endpoint transaksi peminjaman & pengembalian
29. Buat endpoint import CSV
30. Buat endpoint laporan (pakai views dari Fase 3)
31. Buat endpoint metadata dokumen

## Fase 6 — Frontend/Dashboard
32. Init project Vue 3 + Vite + Tailwind
33. Buat halaman login
34. Buat dashboard ringkasan (Chart.js — jumlah buku, peminjaman aktif, dsb)
35. Buat halaman anggota & buku
36. Buat halaman transaksi peminjaman/pengembalian
37. Buat halaman dokumen (list + metadata)
38. Buat halaman matriks akses
39. Embed gambar ERD ke dashboard

## Fase 7 — Backup & Testing
40. Backup database (`mysqldump`)
41. Uji restore dari hasil backup, validasi data konsisten (cek COUNT record sama)
42. Uji seluruh alur end-to-end sesuai poin demonstrasi di DIT

## Fase 8 — Dokumentasi & Presentasi
43. Susun dokumen analisis kebutuhan & permasalahan
44. Susun jawaban FR.AK.03 per KUK tiap unit kompetensi
45. Kumpulkan 10 bukti yang diminta DIT (analisis, ERD, script, query, CSV, laporan kualitas data, metadata dokumen, matriks akses, backup, laporan evaluasi)
46. Susun slide presentasi PowerPoint (ikuti 8 poin fokus presentasi di DIT)
47. Latihan demo sesuai urutan poin demonstrasi di DIT

Saran: kerjain **1 fase penuh dulu sebelum lanjut fase berikutnya** (terutama Fase 1-4 yang murni database, karena itu inti dari 7 unit kompetensi), baru masuk backend/frontend buat kebutuhan demo visual. Mau gw mulai bantu dari Fase 1 (DDL berdasarkan entitas di PRD), atau lo mau coba jalanin sendiri dulu dan balik kalau ada kendala?
